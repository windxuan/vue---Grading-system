一、第一种思路
  1. 管理员登录
  2. 问卷列表 -> 选择问卷
  3. 生成问卷url -> 多份 -> 可查看每项问卷汇总
  4. 每份分发给不同用 -> 用户登录 -> 利用获取的url进行登录
  5. 答题界面
  6. 答题结束 -> 个人汇总界面

二、第二种思路
  1. 用户登录
  2. 问卷列表
  3. 用户选择问卷 -> 生成问卷的url(1份)
  4. 弹出框 询问是否选择该问卷 -> 如果确认,生成答题 -> 否则,停留在原界面
  5. 答题结束 -> 个人汇总界面
  6. 管理员登录
  7. 问卷列表
  8. 点击某一项列表, 出现该问卷的总汇总界面

我们组采用第二种方法 -> 注意事项:
  1. 请求提交时, 如果失败 -> 页面停留原地,成功则跳转。
  2. 答题不符合规范, 采用特殊标识 -> 标识题目
  3. 尽可能帮助用户进行操作 -> 去掉不必要的操作
  4. 注意vue周期
  5. 注意后退操作
  6. 后退时必须存储数据 -> 或清除数据
  7. 未使用注销按钮而退出时，必须能不登录而进入需要登录的页面。

修改大体思路: 
  1. 用户登录 -> 修改 工号-密码-姓名-性别[-出生年月[-邮箱地址]]
  2. 列表界面 -> 跳转出现列表
  3. 选中问卷后, 先生成问卷url 如果url生成成功 -> 出现弹出框,询问用户是否确定登录
  4. 取消,弹出框消失,确认 ,跳转到答题页面->注意请求必须通过才能跳转到答题界面
  5. 答题完毕 跳转到个人汇总页面
